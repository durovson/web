# JAMMM — портфолио для GitHub Pages

Одностраничный адаптивный сайт-портфолио в эстетике исходного макета: крупный hero-блок с котом, секции `01 — ABOUT`, `02 — COLLECTION`, `03 — BOTS`, `04 — BRANDS`, `05 — CONTACT` и бесконечная NFT-лента.

Проект написан на чистых HTML, CSS и JavaScript. Сборка, npm и серверная часть не нужны, поэтому файлы можно редактировать прямо на GitHub и публиковать через GitHub Pages.

## Структура

```text
index.html       — структура страницы, тексты и ссылки
style.css        — палитра, сетки, адаптивность и анимации
script.js        — меню, счетчики, эффекты кота и NFT-ленты
README.md        — эта инструкция
images/
  cat-hero.jpg   — изображение героя
  nft1.webp      — карточки коллекции
  nft2.webp
  nft3.webp
  nft4.webp
```

Все изображения подключены относительными путями `images/...`, поэтому сайт работает и локально, и на GitHub Pages.

## Локальный запуск

Самый простой способ — открыть `index.html` двойным щелчком. Для проверки в режиме локального сайта можно выполнить в папке проекта:

```bash
python -m http.server 4173
```

После этого откройте `http://127.0.0.1:4173/`. Остановить сервер можно сочетанием `Ctrl+C`.

## Как изменить текст и ссылки через GitHub

1. Откройте репозиторий и файл `index.html`.
2. Нажмите кнопку с карандашом `Edit this file`.
3. Найдите нужную секцию по комментарию, например `03 — BOTS`.
4. Измените текст между HTML-тегами, не удаляя сами теги.
5. Для контактов замените и видимый username, и адрес внутри `href`.
6. Нажмите `Commit changes` и подтвердите сохранение.

Пример контактной ссылки:

```html
<a href="https://t.me/username">@username</a>
```

## Как заменить изображение

1. Откройте папку `images` в репозитории.
2. Нажмите `Add file` → `Upload files` и загрузите новое изображение.
3. Откройте `index.html` для редактирования.
4. Укажите имя нового файла в атрибуте `src`, например:

```html
<img src="images/new-cat.png" alt="Рыжий кот за ноутбуком">
```

Лучше использовать JPG или WebP для фотографий и PNG для изображений с прозрачностью. У файлов не должно быть пробелов и кириллицы в названии.

## Как добавить NFT-карточку

В `index.html` найдите `.marquee__group` и скопируйте одну карточку перед закрывающим `</div>` этой группы:

```html
<article class="nft-card">
  <img src="images/nft5.jpg" alt="Название NFT">
  <p>НАЗВАНИЕ <span>005</span></p>
</article>
```

Загрузите соответствующий файл в `images/`. Вручную дублировать карточки не нужно: `script.js` сам создаёт вторую копию всей группы, чтобы анимация зацикливалась без скачка.

## Цветовая палитра

Все цвета собраны в начале `style.css` в блоке `:root`:

```css
--orange: #c75b00;
--apricot: #f89f4b;
--ink: #68614e;
--brown: #9c795b;
--olive: #95815e;
--grey: #7f7573;
--cream: #d4c1a1;
```

Меняйте значения только в этом блоке — остальной сайт обновится автоматически.

## Публикация на GitHub Pages

1. Загрузите `index.html`, `style.css`, `script.js`, `README.md` и папку `images` в корень репозитория.
2. Откройте `Settings` → `Pages`.
3. В `Build and deployment` выберите `Deploy from a branch`.
4. Выберите ветку `main` и папку `/ (root)`.
5. Нажмите `Save`.
6. Через несколько минут ссылка появится в этом же разделе Pages.

Если главная страница не открывается, проверьте, что файл называется именно `index.html` и находится в корне выбранной ветки.

## Что делает JavaScript

- открывает и закрывает мобильное меню;
- подсвечивает текущую секцию в навигации;
- показывает блоки при прокрутке;
- запускает счетчики статистики;
- добавляет scramble-эффект заголовку;
- автоматически клонирует NFT-группу для бесшовной ленты;
- слегка наклоняет баннер с котом вслед за курсором.

При системной настройке «уменьшить движение» интенсивные эффекты отключаются автоматически.
